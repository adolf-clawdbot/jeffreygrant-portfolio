Overview

The Challenge

Amazon's fulfillment operations relied on three disconnected systems: warehouse associate tools, Seller Central dashboards, and operations manager analytics. This fragmentation created inefficiencies, with FC associates spending 18% of their shift navigating clunky interfaces, sellers unable to track real-time inventory, and ops managers manually reconciling data from multiple sources.

Project Meridian aimed to unify these experiences into a single, cohesive portal built on Amazon's Cloudscape Design System.

Design Process

Building the Solution

Key Design Decisions

1. Role-Based Dashboards — Different interfaces for associates (mobile-first, high contrast), sellers (desktop analytics), and managers (reporting tools)

2. Real-Time Data Sync — Inventory updates every 15 minutes instead of 6+ hours, reducing seller anxiety and stockout incidents

3. Cloudscape Design System Extension — Built 14 custom React components on top of Amazon's open-source Cloudscape library for consistency across Amazon products

Design System Components Created

  • InventoryHealthRing — Visual stock level indicator with color-coded zones
  • ShipmentTimeline — Step-by-step progress tracker for inbound shipments
  • PickTaskCard — Mobile-optimized associate task display with large touch targets
  • AlertPriorityBadge — P1-P4 severity indicators for urgent actions
  • FCLocationPicker — Multi-select warehouse filter with autocomplete

Reflection

Key Learnings

What Worked

Early FC Shadowing: Spending 3 full shifts at BFI4, SDF8, and JFK8 revealed critical usability issues with handheld scanners that we wouldn't have caught in lab testing.

Iterative Prototyping: We tested 4 dashboard layouts with 80+ users before finalizing. The winner wasn't the team's favorite — but it was the users' clear choice.

What I'd Do Differently

Stakeholder Alignment: We should have involved FC ops leaders earlier. We redesigned the manager reporting module twice because we didn't understand their compliance requirements upfront.

Mobile-First from Day One: We initially designed for desktop and retrofitted mobile. The next iteration will start with the associate mobile experience and scale up.
